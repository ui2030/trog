# 🛡 장비 4슬롯 시스템 도입 (2026-04-28)

## 변경 요지

기존 3슬롯 (`weapon` / `armor` / `accessory`) → **4슬롯**
- `main_hand` (왼손) — 주무기. 검·창·지팡이·완드 등.
- `off_hand` (오른손) — 보조. 방패·보조무기 또는 양손 무기의 한쪽.
- `armor` (방어구) — 통합 슬롯. 갑옷·로브·투구·부츠·장갑·장화 모두 여기.
- `accessory` (장신구) — 반지·목걸이·부적·성표·마법서.

**dual-wield 자동 감지** — 이름에 `쌍단검`/`쌍검`/`듀얼 대거` 포함 시 양손 슬롯 모두 동일 아이템 자동 장착. UI 에선 "양손 (쌍)" 으로 통합 표시.

## 직업별 기본 장비 변경

| 직업 | 왼손 (main_hand) | 오른손 (off_hand) | 방어구 | 장신구 |
|---|---|---|---|---|
| 전사 | 녹슨 장검 | **낡은 방패** ← (이전 accessory) | 가죽 흉갑 | (비움) |
| 마법사 | 견습생의 지팡이 | (비움) | 수련자 로브 | 작은 마법서 |
| 도적 | **단검 (왼손)** | **단검 (오른손)** ← 쌍단검 | 어두운 가죽 갑옷 | 도둑의 밧줄 |
| 성직자 | 축복받은 철퇴 | **낡은 방패** ← (이전 accessory) | 성스러운 사제복 | 성표 |

## 변경 파일

### `trog/main.py`
1. **CLASS_STATS** (4곳) — `weapon` 제거, `main_hand`/`off_hand` 추가
2. **Player.__init__** — equipped 딕트 4슬롯 초기화
3. **ITEM_KIND_KEYWORDS** — 키워드 확장: `왼손`, `오른손`, `방패`, `반지`, `목걸이`, `부츠`, `투구`, `장갑` 등
4. **_NAME_SLOT_HINTS** — 정규식 슬롯 분류 강화:
   - `쌍단검|쌍검|쌍도|듀얼` → `dual` (양손)
   - `방패|실드|버클러` → `off_hand`
   - `갑옷|로브|투구|신발|부츠|장갑|망토` → `armor`
   - `반지|목걸이|부적|성표` → `accessory`
   - `검|도|창|활|단검|도끼|망치|지팡이|완드` → `main_hand`
5. **parse_and_apply_items** — `slot == "dual"` 분기 신설, 양손 동시 장착
6. **_SLOT_ALIASES** — 해제·매칭용 별칭 확장 (한↔영, 신↔구)
7. **Player.from_save_dict** — 구 save (`weapon` 슬롯) → `main_hand` 자동 마이그레이션
8. **DM_SYSTEM_PROMPT** — 4슬롯 시스템 + 쌍단검 dual-wield 규칙 명시
9. **_players_summary** — DM 향 파티 요약을 `왼손:X / 오른손:Y / 방어구:Z` 포맷으로 (양손 동일 시 `양손:X` 통합)

### `trog/static/game.js`
1. **refreshCharPanel** equipment 섹션 — 4슬롯 (또는 dual 시 3슬롯) 렌더
2. **takeover modal** — `eq.main_hand` 우선 표시 (구버전 `eq.weapon` 폴백 + 딕트 .name 추출)

## 자동 감지 예시

| DM 입력 | 자동 분류 |
|---|---|
| `[A 획득: 강철 방패 \| 무기 \| 방어 +3]` | 이름에 "방패" 매치 → **off_hand** (이전엔 weapon 으로 잘못 등록됨) |
| `[A 획득: 강철 부츠 \| 무기 \| 회피 +2]` | 이름에 "부츠" 매치 → **armor** (이전엔 weapon 으로 잘못 등록됨) |
| `[A 획득: 신비의 반지 \| 장신구 \| INT +2]` | "반지" → **accessory** (정상) |
| `[A 획득: 쌍단검 \| 무기 \| 2회 공격]` | "쌍" 매치 → **dual** → main_hand + off_hand 양쪽 |
| `[A 획득: 강철 검 \| 무기 \| 공격 +5]` | "검" → **main_hand** (왼손) |

## 비판적 셀프 리뷰

### 🟢 호환성
- 구 save 파일에 `equipped['weapon']` 있으면 자동으로 `main_hand` 로 마이그레이션
- `_SLOT_ALIASES` 가 `weapon` 도 받아들여 해제·태그 매칭 모두 호환
- 클라 폴백 `eq.main_hand || eq.weapon` 으로 표시도 안 깨짐

### 🟡 한계
- **방어구 통합 슬롯** — 갑옷·부츠·헬멧·장갑 모두 한 슬롯 (armor) 차지. 이미 갑옷 입은 상태에서 부츠 주우면 이전 갑옷이 인벤으로 회수됨. UX 손실. 향후 `helmet`/`boots`/`gloves` 별도 슬롯 분리 가능하지만 복잡도 ↑.
- **DM 의 `종류 칸` 인지 한계** — 프롬프트 강화했지만 LLM 이 가끔 부츠를 "무기" 종류로 찍을 수 있음. `_NAME_SLOT_HINTS` 가 이름 매치로 교정해주지만 이름이 모호하면 (예: "신비의 룬 아이템") 잘못된 슬롯 가능.
- **dual-wield 단일 보유** — 쌍단검 한 자루를 양손에 표시하지만 내부적으로 한 인벤 항목. 만약 한 손만 잃어버리는 상황(예: 한 단검을 던졌다)이 서사상 발생하면 표현 불가. 단순화 trade-off.
- **양손 무기 vs 쌍** — `쌍단검` 은 dual, 그러나 `대검` 은 양손 무기지만 main_hand 에만 들어가고 off_hand 는 비어있음. UI 상 차이 없어 헷갈릴 수 있음. 미세한 한계.

### 🟢 보안
- `_classify_item` + `_correct_slot_by_name` 조합으로 LLM 잘못된 슬롯 지정 자동 교정
- 잘못된 입력값 (`slot == "dual"` 외 + dual 외) 은 기존 검증 흐름 그대로 거부
