## 새 기능 — 창의력

### 1. XP / 레벨업 자동화
- **LLM 태그:** `[이름 XP +N]` (전투 승리 20~40, 일반 기여 5~15, 대담한 한 수 50+)
- **임계값** ([main.py](trog/main.py) `xp_needed_for`): Lv2=100, Lv3=250, Lv4=450, Lv5=700, Lv6=1000 …
- **레벨업 보상:** `max_hp +10`, `attack +2`, 현재 HP 풀회복 — **극적 순간 연출**
- **UI:** 보라→골드 그라디언트 XP 바, 레벨업 토스트, 캐릭터 패널 골드 플래시 1.6초
- **파티 요약에 레벨 노출** → DM이 "Lv3 전사"를 인지하고 난이도/톤 조절 가능

### 2. 아이템 인벤토리
- **LLM 태그:** `[이름 획득: 아이템명]`
- 서버가 `Player.inventory: List[str]`에 추가, 중복 자동 필터
- **캐릭터 패널에 소지품 칩** (골드 outline 칩)
- **파티 요약에 최근 3개 노출** → DM이 후속 서사에 반영 ("철검을 쥔 허접은…")
- 40자 초과 아이템명은 오탐으로 판단해 drop

### 3. 커스텀 퀵액션
- 기본 5개 버튼(탐색/대화/공격/치료/매복) 아래 사용자 정의 행동 최대 6개
- `＋ 나만의 행동` 버튼 → 라벨 + 이모지 + 실제 보낼 텍스트 prompt 3단계
- 우클릭으로 삭제
- localStorage 저장 (`trog-custom-actions`)
- **도적이 "은신 기습" 버튼 만들어 두고 반복 사용**, **성직자가 "축복" 상용구 저장** 같은 플레이 패턴 촉진

### 4. "Yes, but... / No, and..." 창의력 프롬프트
[main.py DM_SYSTEM_PROMPT](trog/main.py#L164-L201) 추가:
> "플레이어가 특이하거나 엉뚱한 시도를 하면 단순히 실패로 치지 말고 흥미로운 결과를 만들어라. 판정이 실패해도 이야기가 전진하게 하라."

전통 TRPG의 즉흥 GM 원칙. 실패가 막다른 길이 아니라 새 전개의 씨앗이 되게.

## 새 기능 — 편의

### 5. 자동 캐시버스터
- **기동 시각 기반** `STATIC_VERSION = str(int(time.time()))`
- `/` 라우트가 index.html 읽을 때마다 `?v=\d+` 전부 `?v={STATIC_VERSION}`으로 치환
- **수동 `?v=2 → ?v=3` bump 불필요** — V2에서 🟡로 지적한 항목 해소
- HTML 파일 수정해도 서버 재시작만 하면 브라우저가 새 리소스 로드

### 6. 레벨업/XP/아이템 토스트
- 우상단 오버레이에 3.2초 슬라이드인
- **내 이벤트**는 강조 (골드/굵게), **남의 이벤트**는 조용히
- 레벨업은 따로 강조 toast + 캐릭터 패널 플래시
- 별도 DOM 레이어로 격리 → 기존 narrative log 오염 없음

## 새 기능 — 안전 / 견고성

### 7. 시간 역행 방지
- 시간대마다 ordinal: `{🌅:0, ☀️:1, 🌞:2, 🌆:3, 🌙:4, 🌌:5}`
- DM이 본문 중 과거 시간 언급하며 태그 쓰면 **현재보다 작은 ordinal은 무시**
- V2 🔴에서 지적한 시간 역행 허점 해소

### 8. 레이트리밋
- 플레이어당 3초 쿨다운 (`GameRoom.last_action_at`)
- 초과 시 `⚠ 너무 빠릅니다. 1.4초 뒤에 다시 시도하세요.` 에러 메시지
- **왜:** 버튼 연타로 LLM 호출 폭탄 방지, 구독 토큰 보호

### 9. 방 코드 충돌 방지
- 기존 `uuid.uuid4()[:6].upper()`는 **덮어쓰기 위험** (희박하지만 실존)
- `_new_room_code()` — 기존 방과 겹치면 최대 50회 재시도, 폴백 8자

### 10. 메시지 히스토리 트림
- `MESSAGE_HISTORY_CAP=50`, 매 append 후 슬라이스
- LLM 컨텍스트는 기존대로 최근 20개만 전송
- **장시간 플레이 시 메모리 누수 방지**

### 11. 행동 입력 400자 상한
- 프롬프트 주입 완화 + 토큰 보호
- 서버 측에서 자르고, 200자 이상은 프론트 maxlength에서 1차 컷

## 비판적 셀프 리뷰 — V3에서 안 한 것 / 남은 허점

### 🟡 영구 저장(persistence) 미구현
- 방은 여전히 메모리에만. 서버 재시작 = 캠페인 증발.
- **의도적 보류:** JSON snapshot이든 SQLite든 아키텍처 변경이라 별도 세션으로 다루는 게 맞음. 사용자 확정 후 진행.

### 🟡 LLM이 XP 태그 **남발**할 수 있음
- 프롬프트에 "남발 금지"를 넣었지만 보장 안 됨. 한 응답에 `[허접 XP +40]`이 3번 나오면 120 적립.
- **완화 가능:** 응답당 최대 1회 XP 부여 강제 — 현재는 파서가 for문으로 전부 누적.
- **현재 가드:** `grant_xp` 음수 방어, 아이템 중복 자동 필터.

### 🟡 레벨업 시 풀회복은 강한 보상
- 전투 한복판에서 레벨업 → 즉시 풀회복은 극적이지만 밸런스 붕괴 여지. 플레이해보고 조절.

### 🟡 커스텀 퀵액션에 icon 검증 없음
- prompt로 받는 이모지는 문자열 slice(0,4)만 — 사용자가 "ABC"를 icon으로 넣어도 그대로 노출됨. 큰 문제 아니지만 미학 이슈.

### 🟡 레이트리밋 3초는 경험상 값
- 긴 서사에 짧은 후속 액션(예: "아, 잠깐!")이 자연스럽게 잘릴 수 있음.
- **완화 가능:** 쿨다운을 LLM이 응답한 시점 기준으로 재설정하면 버튼 연타만 잡고 자연 흐름은 방해 안 함.

### 🟡 시간 역행 방지가 "같은 ordinal" 허용
- `🌙 → 🌙` 은 허용 (의도). 하지만 `🌆 → 🌆`만 반복되면 하루 넘기기 어려워짐.
- **다음 개선:** 별도 "day counter" — 같은 ordinal 재등장 + 키워드 "다음날"이면 카운터 증가.

### 🟢 auto cache-buster는 dev/prod 모두 OK
- 서버 재시작 시마다 토큰 변경 → 브라우저 캐시 무효
- prod에서도 문제 없음 (재시작 빈도 낮음 = 캐시 유지, 업데이트 후 = 갱신)
