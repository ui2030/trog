## 비판적 셀프 리뷰

### ✅ 시간 파싱 폴백 추가 (V2 후속 반영)
`TIME_PATTERN`이 이모지 태그 못 찾으면 **키워드 폴백 테이블**로 추정:
- "심야/자정" → 🌌 심야
- "밤/달빛" → 🌙 밤
- "황혼/저녁/노을" → 🌆 황혼
- "정오/한낮" → 🌞 정오
- "아침/오전" → ☀️ 아침
- "새벽/동틀/여명" → 🌅 새벽

LLM이 포맷 안 지켜도 웬만하면 배지 뜸. 검사 범위도 앞 300자로 확대.

### 🟡 이름 톤 반영은 LLM 재량
프롬프트에 지시 추가했지만 **보장 안 됨**. 모델이 무시할 수 있음. Sonnet 4.5는 따르는 경향이 강하지만 Claude Code CLI 모드(wrapper)는 내부적으로 다른 시스템 프롬프트가 강제 주입되므로 덜 민감할 수 있음.

**확인 방법:** "허접"으로 플레이해서 첫 응답 톤 보고 판단.

### ✅ "방 나가기" 버튼 추가 (V2 후속 반영)
char-panel 하단에 `🚪 방 나가기 / 새 캐릭터` 버튼. 클릭 시:
1. confirm 다이얼로그
2. WebSocket close
3. localStorage 클리어
4. 하드 리로드

경고 색상(어두운 빨강)으로 실수 방지. `?fresh=1`은 백업용.

### 🟡 캐시 파라미터 업데이트 수동
수정할 때마다 `?v=2` → `?v=3`으로 올려야 함. 자동화 안 됨.

**대안:** 개발 환경에서는 `Cache-Control: no-store` 헤더 설정. 하지만 FastAPI StaticFiles는 기본 설정 변경이 귀찮음.

### 🔴 DM이 시간 태그를 본문 중간에 쓰면?
예: `"@@ 시간이 흘러 [🌙 밤]이 되었다"` — 이 경우 첫 200자 검사에 포함되면 잡힘. 하지만 LLM이 본문 중 **과거/미래** 시간을 언급하면서 태그 쓸 수도. 드물지만 시간 역행 가능.

**완화책 없음 현재.** 나중에 "이전 시간 > 새 시간 인지 체크" 로직 추가 가능.

### 🟢 `current_time`은 rejoin에도 전달됨
재연결 시 배지 복원됨.
