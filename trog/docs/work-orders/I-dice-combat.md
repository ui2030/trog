## I-1. 서버: 공격 감지 + 판정 (main.py)

1. `_is_attack_action(text) -> bool`: _is_ambush_action(기존) 패턴의 키워드 휴리스틱
   (공격/벤다/친다/쏜다/시전/찌른다 등 — 기존 함수의 어휘 목록 방식 따르기).
   전투 중(몬스터 생존)일 때만 검사.
2. 판정 상수(튜닝 레버, 전부 모듈 상단): `COMBAT_DC_BASE = 10`, 몬스터 티어 보정
   (max_hp 3구간: 약 +2 / 중 +4 / 강 +6 — 구간 경계도 상수), `CRIT_MIN = 19`(natural 19~20 크리).
3. 능력 보정: 직업별 주 능력(전사=근력, 마법사=지능, 도적=기교, 성직자=지혜 — 기존 능력치
   명칭·보정 산식은 탐색 판정이 쓰는 것 그대로 재사용, 새 산식 금지).
4. player_action 처리에서 공격 감지 시 1회 굴림 → 결과 3단(크리/명중/빗나감)을
   pending_notes로 주입. 문구는 탐색 주입 스타일:
   - 크리: "[전투판정] {이름}의 공격: d20 {n}(치명타!)+{보정}={합} vs DC {dc} → 치명적 명중.
     강렬하고 극적으로 서술하고 피해 태그는 평소보다 크게(기존 상한 내)."
   - 명중: "... → 명중. 이 결과대로 서술하라."
   - 빗나감: "... → 빗나감. 공격 실패로 서술하고 몬스터 HP 태그를 쓰지 마라. 적의 반격 여지는 재량."
5. 같은 응답에서 DM이 판정과 모순되게 굴어도 숫자는 기존 클램프(C 패키지)가 지킨다 —
   추가 강제 장치 만들지 마라(빗나감인데 HP 태그가 오면 로그 warning 1줄만).

## I-2. 클라: 주사위 연출 (game.js + style.css)

1. 판정 발생 시 서버가 broadcast `{"type": "dice_roll", who, d20, mod, mod_label, total, dc, outcome}`
   (outcome: "crit"|"hit"|"miss"). 전원에게 — 남의 굴림도 보이는 게 테이블 감성.
2. 클라: 주사위 토스트 — 🎲 + 숫자가 0.6초 정도 빠르게 바뀌다 멈추는 연출(CSS/JS 간단히,
   `prefers-reduced-motion`이면 즉시 최종값), 결과색: 크리=금색, 명중=초록, 빗나감=붉은색.
   "{who} 🎲 {d20}+{mod}({mod_label}) = {total} vs {dc}" 한 줄. 기존 토스트 레이어·캡(6개) 재사용.
3. 탐색 판정은 손대지 않는다 — 이미 pop 카드가 수치를 노출함(중복 연출 금지).
