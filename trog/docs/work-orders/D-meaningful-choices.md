## D-1. 종족 보정 리워크 (main.py)

근본 원인(중요): `apply_race_modifiers`(~3687)는 RACE_ABILITY_MOD 의 **부호만** 쓰고 크기는
매번 `random.randint(1,3)` 이다. 그래서 표의 "+2/+1" 은 장식이고, 항목 수가 많은 인간
(6개 전부 양수)은 기대값 **+12** vs 타 종족 +2 안팎 — 표가 의도한 밸런스가 코드에 없다.

1. `apply_race_modifiers` 를 **면값 적용**으로 교체: `delta = sign_val` 그대로 (랜덤 크기 제거).
   기존 ABILITY_MIN/MAX clamp·1회 적용 가드·applied 반환 형식은 유지.
2. 인간(~668): 6스탯 +1 → **3스탯 +1** 로 축소. 컨셉 "적응력": `{"wisdom":1, "charisma":1, "constitution":1}`.
3. 토끼(~685): `constitution: -1` 제거 → `{"dexterity":2, "wisdom":1}` (순 +3).
4. 결과 순보정 표(전 종족 +2~+4 로 수렴):
   인간+3 / 엘프+2 / 드워프+2 / 하플링+2 / 오크+2 / 티플링+2 / 드래곤본+3 / 놈+2
   / 수인: 늑대+3·여우+4·호랑이+4·고양이+4·토끼+3·곰+3
5. 랜덤 변이 제거로 캐릭터별 편차 재미가 사라지는 건 의도된 트레이드오프(예측 가능한 밸런스 우선) — 코드 주석에 한 줄 남겨라.
6. 종족 선택 UI(index.html)에 보정치 표기가 있으면 새 수치와 일치하는지 확인.

## D-2. 탐색 위험도 차등 (main.py)

문제: 함정은 비살상(최소 1HP)에 최대 10 피해, 골드는 전원 지급 — '상'을 고를 손해가 사실상 0이라 항상 '상'이 정답.

1. 함정 피해 clamp 를 위험도별로: 하/중 = 기존 1~10 유지, **상 = 4~15** (LLM 각본 damage clamp 와 폴백 모두). 비살상(최소 1HP) 규칙은 유지 — 죽음 대신 아프게.
2. 폴백 각본 골드: 상 20~60 → **30~90** (하/중 유지 — 하 5~20, 중 10~35).
3. LLM 각본 프롬프트(generate_exploration_script)에 한 줄 추가: "위험도 상이면 item 중 1개는 희귀 장비(slot 지정: 무기/방어구/장신구 중)로."
4. 기존 탐색 테스트(_test_exploration.py)에 위험도별 함정 clamp assert 1개 추가.

## D-3. 부활 페널티 (main.py)

문제: HP 0 이후 아무 손실 없이 부활 대기 — 죽음이 무섭지 않다.

1. HP 태그로 0 → 양수 부활이 일어나는 경로(HP 태그 파서에서 old_hp==0 and new_hp>0)에서
   서버가 부활 HP 를 `max_hp 의 30%` 로 캡(최소 1). 이 캡은 C-2 회복 clamp 와 별개 규칙.
2. 플레이어 상태효과 시스템이 있는지 확인(몬스터엔 status_effects 존재 — 플레이어 지원 여부 grep):
   - 지원하면: 부활 시 "깊은 부상"(3라운드) 상태효과 부여 — 기존 상태효과 표시/토스트 경로 재사용. 기계 효과(공격 감소 등)는 시스템이 지원하는 범위에서만, 무리한 신규 메커니즘 금지.
   - 미지원이면: 히스토리에 시스템 노트 "[시스템] {이름}은(는) 깊은 부상을 입은 채 깨어났다 — 당분간 무리한 행동은 어렵다." 를 남겨 DM 이 롤플레이로 반영하게. 어느 쪽을 택했는지 결과 보고에 한 줄.
3. 단위 테스트 1개: HP 0 플레이어에 풀피 부활 태그 → 30% 캡 확인.
