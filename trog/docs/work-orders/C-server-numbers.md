# [패키지 C] 서버가 숫자를 쥔다 — 골드·회복·XP·아이템 상한 + 미니 상점

## C-1. 골드 태그 상한 (main.py)

문제: 골드 태그 파서(~1979-2013)에 상한이 없다 — `[이름 골드 +50000]` 이 그대로 적용됨.

- 증감형(`골드 +N/-N`): 1회 태그당 델타를 **±2,000** 으로 clamp.
- 절대설정형(`골드: N`): **0~999,999** clamp.
- 최종 잔액도 999,999 상한.
- clamp 발동 시 logger.warning 한 줄 (남용 모니터링용).

## C-2. 회복 상한 (main.py)

문제: HP 태그(~1961)가 max_hp 까지 무제한 회복 허용 — LLM이 매 턴 풀피 가능, 소모전 붕괴.

- HP 태그로 플레이어가 **회복**되는 경우(new_hp > old_hp): 1회 태그당 회복량을
  `max(1, int(max_hp * 0.4))` 로 clamp. 피해 방향은 기존 그대로.
- 서버가 직접 주는 회복(C-5 물약, 부활 등 서버 발 경로)은 이 clamp 를 우회(태그 파서만 제한).
- 몬스터 회복은 제한 없음(적 자가치유는 DM 연출 영역).

## C-3. XP 재조정 (main.py)

문제: 수동 XP 태그 상한이 헐거워(응답당 500 = Lv1→4 한 방) 성장이 DM 기분에 좌우, 자동 처치 XP는 인색.

- `XP_GAIN_MAX_PER_EVENT` 200 → **100**, `XP_GAIN_MAX_PER_RESPONSE` 500 → **150** (~1882-1883).
- 자동 처치 XP(~4921-4934): `12 + max_hp//4` (clamp 12~80) → **`20 + max_hp//3` (clamp 15~150)**.
  어시스트: kill//3, clamp **5~50**. (HP 272+ 보스 XP 정체도 이 상향으로 완화)
- DM 시스템 프롬프트의 XP 안내 문구가 있으면 새 상한에 맞게 숫자만 갱신.

## C-4. 아이템 지급 상한 (main.py)

문제: `parse_and_apply_items`(~2402)가 응답 내 획득 태그 수 무제한 — 한 응답에 장비 대량 지급 가능.

- 응답당 획득 태그 **3개까지만** 적용, 초과분은 무시 + logger.warning.
- 자동 장착(장비 슬롯 갱신)은 응답당 **1개**로 제한(대장간 경로의 기존 1개 규칙을 일반 서사 경로에도 적용). 초과 장비는 인벤토리로만.

## C-5. 미니 상점 — 서버 고정가 물약 (main.py + game.js + style.css)

문제: 서버가 강제하는 소비처가 없어 골드가 쌓이기만 함(상점/대장장이 전부 LLM 서사 의존).

1. 카탈로그(서버 상수, LLM 무관): 회복 물약 60G(HP+40) / 고급 회복 물약 150G(HP+100) / 마나 물약 60G(MP+40).
2. 새 WS 메시지 `shop_buy {item_key}`: 게임 시작 후 언제든 허용(사망자 제외). 골드 검증→차감→기존 아이템 지급 파이프라인(합성 태그 재사용 가능)으로 인벤토리에 추가. 잔액 부족은 에러 토스트.
