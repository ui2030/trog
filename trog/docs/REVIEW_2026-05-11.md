## ⚠ 잠재 이슈 (셀프 리뷰)

1. **`_save_deleted` 의 60초 윈도우가 충분한가?** 정상 디바운스(2초) + to_thread 큐(<<1초) 면 충분. 하지만 머신이 IO bound 로 잠겨 있어 60초 넘어가면 race 다시 가능. 모니터링 필요 — 현재로선 logger 가 phantom 케이스 자체를 잡지 못함. 향후 `[SAVE PHANTOM]` warning 추가 검토.
2. **HP/XP `was_dead` 부활 차단** — DM 이 `[허접 HP: 0 → 30]` 으로 명시적 부활을 시도해도 그 줄은 `parse_and_apply_hp` 가 정상 처리. `grant_xp` 만 차단하므로 의도된 부활(체크포인트, 부활 주문)은 영향 없음. ✅
3. **off_hand 회수 fallback** — `grant_item` 이 인벤 cap 초과로 실패할 케이스가 있다면 silent loss. `grant_item` 의 반환값이 False 여도 try/except 가 무시. cap 도달은 매우 드문 시나리오라 일단 이대로.
4. **gold delta 보고 변경**으로 클라이언트 토스트 표기가 살짝 달라짐 — 이전엔 `−100` 표기였으나 잔액이 `−50` 만 변했다면 이제 `−50` 으로 표시. UX 적으로 더 정확하지만 기존 토스트 톤에 익숙한 플레이어는 살짝 어색할 수 있음.
5. **rate-limit cancel 환급 1초** — UX 영향. 현재 cancel 후 즉시 재시도하던 사용자는 이제 1초 쿨다운. 만약 LLM 호출 폭탄이 실제 문제가 아니라면 (단일 사용자 환경) 이 변경이 약간 답답할 수 있음. 분리 가능한 옵션이라 향후 env var 화 검토.
6. **VSCode 로 `.bat` 재저장 시 인코딩 다시 깨짐** — 별도 문서 [BAT_ENCODING_FIX.md](BAT_ENCODING_FIX.md) 에서 이미 경고 중. 변동 없음.
7. **`saves/` 가 .gitignore 에 추가됨** — 만약 의도적으로 일부 save 를 commit 하던 워크플로우가 있었다면 `git add -f` 필요. 현재 git 미사용이라 해당 없음.
