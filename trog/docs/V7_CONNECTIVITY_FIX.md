# V7 — 디테일 연결성 재검사 (2026-04-28)

> 사용자 피드백: "디테일적인 연결성이 부족하다 — 사전에 이런 일이 일어나지 않도록 연결성 재검사를 통해 다시 이어줘."

게임 시스템 자체는 거의 다 갖춰져 있는데, **DM 서사가 시스템 태그를 빠뜨려서** 서사와 상태가 어긋나는 케이스가 5건 보고됨. 새 기능을 추가하기보단 **기존 기능의 적용을 보장**하는 방향으로 수정.

---

## 보고된 5개 끊김

| # | 증상 | 원인 |
|---|---|---|
| 1 | 쌍단검 사용 중인데 공격 2회가 안 됨 | DM 프롬프트가 `2회 공격` 효과 텍스트만 박아두고 **"d20 두 번 굴려라"** 명시 안 함 → DM 이 한 번만 굴리고 마무리 |
| 2 | 퀘스트 클리어·보상 받았는데 퀘스트 아이템 인벤에 그대로 | DM 이 "건네줬다" 묘사만 하고 `[이름 사용: 퀘스트아이템]` 태그 누락. 시스템적으로는 사용 태그가 quest 종류도 처리 가능했음 |
| 3 | 방패 장착해도 막기 판정·피해 반감 없음 | 방패 효과 수치(`방어 +N`) 는 자동 합산되지만 **"막는다"** 라는 메커니즘 자체가 프롬프트에 없었음 |
| 4 | 강화 후 무기 이름·효과가 그대로 | 강화 흐름은 본래 `[장비 해제: weapon]` + `[획득: 강화 단검 \| 무기 \| ...]` 두 줄이어야 했는데 DM 이 서사로만 처리 → 슬롯 변동 없음 |
| 5 | 상점에서 무기 구매 → 골드만 빠지고 인벤에 없음 | DM 이 `[골드 -N]` 만 찍고 `[획득:]` 누락 (가장 흔한 desync) |

공통: **DM 이 서사로 풀어버리고 시스템 태그를 안 찍는다**. 시스템은 멀쩡한데 서사와 상태가 분리됨.

## 적용한 수정

### 🔧 시스템 측 (1건)

**`[이름 장비 강화: 슬롯 \| 새 이름 \| 새 효과]` 신규 태그 추가**

기존 강화 흐름이 두 줄이라 DM 이 절반만 찍는 사고가 잦았음. **한 줄 atomic 교체** 태그를 신설:

- [main.py `EQUIP_UPGRADE_PATTERN`](../main.py) — 정규식 (1465 부근, EQUIP_UNEQUIP_PATTERN 직후)
- [main.py `parse_and_upgrade_equipment`](../main.py) — 파서 (1995 부근, parse_and_unequip 직후)
- [main.py `_parse_all_tags`](../main.py) — 호출 추가, return dict 에 `equipment_upgrades` 키 노출

**동작**:
- 슬롯의 `name`/`effect` 를 새 값으로 즉시 교체
- 이전 장비는 **인벤 회수 X** (강화는 같은 무기의 변형 — 별개 아이템 아님)
- 빈 슬롯엔 적용 안 됨 (강화 대상 없음 — `[UPGRADE SKIP]` 로그)
- **dual-wield 자동 동기화**: `main_hand` 강화 시 `off_hand` 가 같은 prev_name 이면 둘 다 갱신
- `new_effect` 빈 문자열이면 기존 효과 보존 (단순 리네임 케이스)

**클라**: [game.js `events.equipment_upgrades` 토스트](../static/game.js) — `⚒ 뮤즈록 왼손 (양손) 강화: 녹슨 단검 → 강화 단검` 식.

players 페이로드는 매 dm_response 마다 `[p.to_dict() for p in players.values()]` 로 다시 보내지므로, 캐릭 패널의 슬롯 이름은 **별도 코드 변경 없이** 자동으로 새 이름으로 갱신됨.

### 📝 DM 프롬프트 측 (4건 + 1건)

[main.py `DM_SYSTEM_PROMPT`](../main.py) — "장비 해제" 섹션 직후에 5개 가이드 추가:

1. **🆕 V7 장비 강화 / 업그레이드 / 리네임** — 새 태그 사용법 + 흔한 실수 + 양손 동기화 안내
2. **🆕 V7 양손 무기 dual-wield 공격 판정** — d20 두 번 굴리고 각각 묘사. 두 번 다 명중 / 한 번 / 둘 다 빗맞 케이스별 결과 분기
3. **🆕 V7 방패 막기 판정** — d20 추가 굴림. ≥15 = -80%, ≥10 = -50%, <10 = 정상. 마법·범위·측면은 막기 어려움
4. **🚨 V7 절대 규칙 — 상점·거래 양쪽 태그** — `[골드 -N]` + `[획득:]` 동시. 한쪽만 찍으면 desync 명시
5. **🆕 V7 퀘스트 완료 인벤 제거** — `[이름 사용: 퀘스트아이템]` + 보상 태그 세트. 건네는 장면 묘사 + 사용 태그가 흔한 누락 포인트

## 비판적 셀프 리뷰

### 🟢 안전성 검증

| 항목 | 결과 |
|---|---|
| `EQUIP_UPGRADE_PATTERN` 이 다른 파서와 충돌? | ❌ — HP/MP/XP/골드/획득/사용/장비해제/버프 모두 다른 키워드. 충돌 없음 |
| 빈 슬롯에 강화 태그 들어오면? | 무시 + 로그. 인벤만 부풀리는 사고 차단 |
| 효과 빈 문자열일 때? | 기존 효과 보존 (덮어쓰기 방지) |
| 정규식 길이 폭주 가능? | `[^\]]+?` 와 `[^\]\|]+?` 로 제한. 닫는 `]` 이전까지만 |
| dual-wield 양쪽 각각 강화 시? | 멱등성 — 첫 번째에서 둘 다 동기화, 두 번째는 변화 없음 |
| `kind == "quest"` 인 아이템에 사용 태그? | 기존 `use_item` 이 종류 검사 안 하고 인벤 차감만 함 → quest 도 정상 차감 |

### 🟡 LLM 의존 한계 (보장 불가)

다음은 **시스템이 강제할 수 없고 DM 모델 컴플라이언스에 달림**:

- **양손 무기 d20 두 번**: 모델이 한 번만 굴리고 끝낼 수도 있음. 프롬프트가 절대 규칙으로 명시했지만 100% 보장 X.
- **방패 막기 판정**: 모델이 그냥 정상 피해로 묘사할 수도. 특히 빠른 전개 우선시할 때.
- **상점 거래 양쪽 태그**: 골드 차감만 찍거나 획득만 찍는 케이스가 가장 잦음. 절대 규칙으로 강조했으나 누락 시 자동 복구 메커니즘 없음.
- **퀘스트 완료 사용 태그**: "건네줬다" 만 적고 태그 누락 가능.

→ **어긋남이 보이면 DM 프롬프트 가이드를 더 단호하게 (`반드시`, `금지`, `❌` 강조)** 강화하는 방향. 또는 LLM 의 동일 컨텍스트를 준 채 후처리로 강제할 수도 있으나(예: 거래 키워드 감지 → 검증), 거짓양성 위험 있어 보류.

### 🔴 새로 보이는 잠재 이슈

1. **DM 프롬프트 길이 증가** — V7 추가로 ~80줄 이상 증가. prefill 시간·토큰 비용 영향. 추후 모델 변경 시 토큰 한도 재점검 필요.
2. **강화 태그가 효과 슬롯 비울 수 있음** — 위에서 보존 처리했지만, DM 이 의도적으로 효과 제거 강화(저주 부여 등)를 하고 싶을 때 우회로가 없음. 필요하면 `[X 장비 효과: 플레이어 \| 장비 \| 효과]` 로 별도 갱신해야 함.
3. **dual-wield 동기화는 main_hand 강화에만** — off_hand 만 강화하면 한쪽만 변경 (= dual-wield 해제와 같은 의미). 현재는 의도된 동작이지만, "한쪽만 다른 무기로 바꾸는 강화" 시나리오를 DM 이 자주 요구하지는 않을 듯. 발생하면 그때 재검토.
4. **이전 V5/V6 한계 그대로** — 자해 디버프, 종족별 발성, 인원 난이도 스케일링 등 기존 LLM 톤 가이드는 V7 에서 손대지 않음.
