# TROG 세션 진행 기록 — 미래의 내가 잊지 않도록

이 문서는 **대화 세션에서 고친 것 / 안 건드린 것 / 의사결정 이유** 를 정리한 운영 로그다.
CHANGELOG 가 버전별 릴리스 노트라면, 이건 **"왜 그렇게 결정했나"** 의 메모.

---

## ✅ 추가·수정된 것들 (세션 전반)

### LLM / 속도
- **reasoning 차단** — `extra_body.chat_template_kwargs.{thinking,enable_thinking}=False` 로 hybrid 추론 모델의 사고 단계 off. 상세: [LLM_REASONING_NOTE.md](LLM_REASONING_NOTE.md)
- **`[REASONING PROBE]` 진단 로그** — wrapper 첫 호출 한 번만 `model`, `elapsed`, `→ OFF|ON` 출력. 추론 차단이 먹혔는지 즉시 확인 가능 ([main.py `_probe_reasoning`](../main.py))
- **max_tokens 튜닝** — 액션 600 / 인트로 700. (350 으로 줄였더니 한국어 토큰/글자 비율 오산으로 문장 잘림 → 600 이 안전선)
- **truncation 감지 + 종결 경계 트림** — `finish_reason='length'` / `stop_reason='max_tokens'` 잡아서 `_trim_to_complete_sentence` 로 반토막 문장을 마지막 강한 종결자(`. ! ? …`)까지 정제. 미완 `[...` 태그도 같이 절단. 따옴표 밸런스 체크.
- **LLM_TIMEOUT_SEC=180** (`.env`) — NIM 콜드 스타트 대비
- **모델 여정**: `z-ai/glm-5.1` (NIM 호스팅 초당 5토큰) → `moonshotai/kimi-k2-instruct-0905` (한국어 품질 낮음) → `qwen/qwen3.5-122b-a10b` (현재)

### 게임 시스템
- **몬스터 `status_effects`** — Player 와 동일 구조. `[적 버프/디버프: ...]` 태그 파싱 + 라운드 종료 시 DOT 자동 HP 감소 + 상태 만료 틱. 처치되면 마지막 공격자에게 XP 분배.
- **자동 처치/어시스트 XP** — 몬스터 HP 변화 시 행동자를 `attackers` 에 기록. HP 0 되면 마지막 공격자=처치자(`kill_xp = 12 + max_hp//4`), 나머지=어시스트(`kill_xp/3`). DM 의 `[XP +N]` 태그와 독립 가산.
- **고아 `[XP +N]` 복구** — DM 이 이름 없이 `[XP +35]` 찍어도 현재 행동자에게 자동 귀속. 이전엔 증발 + 본문 노출.
- **장비 해제 태그** — `[이름 장비 해제: weapon|armor|accessory]` (한국어 슬롯명도 수용: `무기/방어구/장신구`). 무기 투척·파괴·분실 시 UI 반영.
- **버프 수치 모디파이어 표시** — 클라가 `status_effects[].effect` 에서 `공격력 +10` 같은 숫자 파싱해서 스탯 패널에 `공격 15 (+10)` 로 병기. 서버 base stat 은 유지.
- **레벨업 풀회복 금지 → 증가분만 가산** (이전 세션에서 된 것, 유지)

### UX / 클라이언트
- **메타 태그 본문 제거** — HP/MP/XP/고아XP/획득/사용/장비해제/버프/디버프/적(6종)/장비·아이템 효과/플레이어 주사위 — 총 10+ 종류. 이전엔 HP/XP/획득 3종만 걸러서 많은 태그가 본문에 노출됐음.
- **액션 오른쪽 이미지 = 플레이어 프로필** — 이전엔 Pollinations AI 가 액션 키워드 기반으로 다른 캐릭터 그려냈음. 이제 `actor.portrait_url` 그대로 표시. 커스텀 그림도 바로 보임.
- **프로필 재저장 버그** — 해시 source 가 데이터 URL 앞 256자만 써서 JPEG 헤더 동일한 경우 `?v=<hash>` 충돌. 전체 data URL 해시로 변경.
- **캔버스 지우개 버그** — 지우개 활성 상태에서 색 클릭하면 tool 이 안 바뀌어 혼란. 색/RGB 선택 시 자동 `setTool('brush')` 복귀.
- **스포이드 툴 추가** — 💧 버튼. 캔버스 픽셀 클릭 → 색 추출 → brushColor 세팅 → 붓 모드 자동 전환.
- **수인 프로필 비율 반영 강화** — Flux 프롬프트의 앞쪽 지배력 원리를 고려해 비율별 시각 키워드를 맨 앞에 배치. 직업 프롬프트에서 얼굴 표현 어구(`face/eyes/expression/beard`) 를 필터링해 수인 트레이트가 덮이지 않게.
- **몬스터 카드에 status chips 렌더** — 플레이어와 같은 `renderStatusChips` 재사용.
- **토스트 이벤트 확장** — kill/assist XP, DOT tick, 적 처치, 장비 해제, 몬스터 buff/debuff.

### DM 시스템 프롬프트 수정
- **수인 비율 표기 금지** — DM 이 닉네임 앞에 "90% 호랑이 XX" 식으로 붙이지 않게. 숫자/동물명은 참고용이고 서사엔 자연스럽게 녹여야 함.
- **d20 결과-서술 일치** — d20:1 찍고 "명중!" 서술 금지. 주사위가 서사를 지배.
- **XP 이름 필수** — `[XP +N]` 대신 `[이름 XP +N]`. 빠지면 서버가 행동자로 추측 복구.
- **장비 해제 태그 명시** — 무기 던지기/파괴/분실 시 `[이름 장비 해제: weapon]` 필수. 태그 없으면 UI 에 여전히 장착으로 표시돼 모순.
- **상태이상 서술 → 태그 의무화** — 독/마비/출혈/화상 키워드 서술하면 반드시 `[적 디버프: 이름 | 효과 N턴 | 매 턴 -N HP]` 동반. 서술만 하면 서버가 모르고 HP 안 깎임. 아이템(독 단검 등) 효과도 포함.

## ❌ 의도적으로 안 넣은 것 / 롤백한 것

- **스트리밍 응답** — 체감 속도 개선에 가장 효과적이지만 WS 프로토콜·클라 렌더 대공사라 보류. 모델 자체 속도 문제를 먼저 해결하는 방향.
- **버프 숫자로 base stat 수정** — 서버의 `player.attack` 자체를 버프로 올리면 레벨업·저장·토스트까지 연쇄 영향. 위험해서 UI 표시 오버레이만 구현. 서사 판정은 여전히 DM 이 수동.
- **부분 이름 매칭 복원** — 과거에 `_match_player` 부분 매칭이 '철수' vs '김철수' 오탐 유발했음. 정확 매칭 유지.
- **자동 디버프 추론 from 서사** — "독이 퍼진다" 같은 자유 텍스트에서 자동 태그 생성은 브리틀. 대신 DM 프롬프트에서 태그 의무화로 접근.
- **Auto-strip `[XP +N]` 태그 숨김 only** — 숨기기만 하면 XP 증발. 복구 + 본문 제거 함께.
- **시스템 프롬프트 대폭 축소** — 3K 토큰 시스템 프롬프트를 줄이면 prefill 빨라지지만, 정확한 태그 포맷/수인 규칙/몬스터 관리 등이 얽혀있어 섣불리 손대면 DM 컴플라이언스 저하. 현상 유지.

## 📚 운영 상식 / 주의사항

### NVIDIA NIM
- **무료 티어**: 40 req/min, 1000 크레딧 초기 지급
- **모델 ID 네이밍 주의** — docs.api.nvidia.com URL 슬러그는 대시, 실제 API `model=` 값은 점:
  - URL: `qwen-qwen3-5-122b-a10b`
  - API: `qwen/qwen3.5-122b-a10b` ← 이거
- **hybrid reasoning 모델 식별법**: build.nvidia.com 모델 페이지에 `Reasoning ON/OFF` 토글이 있으면 추론 모델 → `chat_template_kwargs.enable_thinking=False` 필수
- **z-ai (Zhipu GLM 계열)** — NIM 에서 호스팅 자원 부족으로 심하게 느림. 회피.

### Korean LLM 품질 서열 (이번 세션 경험)
1. Anthropic Claude Sonnet 4.6 — 모국어급 (유료)
2. Qwen 3.5 — 한국어 공식 강력 지원 언어에 포함 (무료/NIM)
3. Kimi K2 Instruct — 문법·조사 오류 빈번 ("내가랑 코", "멀뚱말고")
4. GLM (z-ai) — Kimi 와 비슷, 추가로 NIM 호스팅 느림

### 한국어 토큰 비율 (Kimi/Qwen 기준)
- 1글자 ≈ 1.4~1.8 토큰
- 350자 제한 → 약 490~630 토큰 필요
- 시스템 프롬프트 "N자 이내" 를 `max_tokens` 로 옮길 때 **여유 50% 이상** 확보

## 🟡 현재 알려진 빈틈 (다음 세션 숙제)

- **DM 태그 컴플라이언스**: 수인 비율 표기, 독 디버프 태그 등 프롬프트로 강제해도 모델이 못 따를 때가 있음. Qwen 로 바꾼 뒤 개선 정도 관측 필요.
- **스트리밍 도입**: 응답 지연 체감을 근본 해결하려면 필요.
- **Prompt caching**: NIM 무료 티어는 미지원. Anthropic 모드 전환 시 자동 활용 가능.
- **`_distribute_kill_xp` 공식 튜닝**: `12 + max_hp/4` 가 체감 짜다는 피드백 있음. 데이터 더 모으고 조정.
- **전투 전 몬스터 카드 노출**: DM 이 `[적 등장]` 찍으면 바로 카드 뜸. "관찰/탐지 성공 시에만 노출" 같은 지연 가능하나 복잡도 증가.
