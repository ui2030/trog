# 탐색 모드 1단계 구현 스펙 (2026-07-12, Fable 기획)

확산성 밀리언아서류 "탐색" 미니게임을 TROG에 이식한다. 핵심 원칙 3줄:

1. **DM이 문을 연다** — 탐색 개방 권한은 DM 태그 하나뿐. 플레이어 버튼은 DM에게 보내는 일반 행동일 뿐이다.
2. **보상은 문 열릴 때 이미 봉투에 담겨 있다** — 구간 진입 시 LLM 1회 호출로 이벤트 각본을 전부 확정. 탭은 각본 재생일 뿐, 탭당 LLM 호출 0회.
3. **기존 시스템에 얹는다** — 태그 파서·아이템 지급·장면 이미지·브로드캐스트 경로를 전부 재활용. 새 인프라 금지.

---

## 1. 새 DM 태그

```
[탐색: <장소명> | <N>칸 | 위험도 <하|중|상>]
```

- 예: `[탐색: 폐허가 된 성채 | 12칸 | 위험도 중]`
- 칸수·위험도는 생략 가능. 기본값: 10칸, 위험도 중. 칸수는 6~16으로 clamp.
- 파서는 기존 태그들과 같은 스타일로 (`\s*` 관용, trailing space 허용 — REVIEW_2026-05-11의 `\s*\]` 교훈 준수).
- 표시 텍스트에서는 태그 제거 (`strip_scene_tag` 패턴 참고).
- **무시 조건**: (a) 이미 탐색 진행 중, (b) 생존 몬스터 존재(전투 중), (c) 방이 게임 시작 전.

## 2. 서버 흐름

### 2-1. 탐색 시작 (태그 감지 시)

DM 응답 후처리 체인(기존 `parse_and_apply_*` 호출부, `main.py` ~6104 부근 `extract_scene_payload` 근처)에 `parse_exploration_tag` 추가.

1. 방 상태에 저장: `room.exploration = {place, danger, cells: [...], pos: 0, active: True, started_at, last_tap_at: {}}`
2. **이벤트 각본 생성** — `llm_complete` 1회 호출 (max_tokens 절약, JSON만 요구):

```json
{
  "scene_en": "ruined castle courtyard at dusk, mist",
  "cells": [
    {"type": "flavor", "text": "무너진 기둥 사이로 바람이 분다."},
    {"type": "empty"},
    {"type": "item", "name": "녹슨 철검", "slot": "weapon"},
    {"type": "gold", "amount": 15},
    {"type": "trap", "text": "발밑이 꺼진다!", "damage": 5},
    {"type": "enemy", "name": "굶주린 구울", "hp": 40}
  ]
}
```

   - 생성 프롬프트 규칙: 정확히 N칸, 이벤트 칸(item/gold/trap/enemy)은 3~5개, **enemy는 최대 1개**, 나머지는 flavor/empty. 위험도별 스케일: 하=enemy 없거나 약함·드랍 소소 / 중=균형 / 상=enemy 강함·드랍 좋음.
   - **서버 측 검증·수리 필수**: 칸수 불일치 → 자르거나 empty로 채움. enemy 2개 이상 → 첫 번째만 유지. damage는 1~10 clamp. JSON 파싱 실패 →
   - **폴백 각본**: LLM 실패 시 내장 랜덤 테이블로 즉석 생성 (flavor 문구 풀 + 소액 gold + 잡템). 탐색 시작이 LLM 때문에 막히는 일은 절대 없어야 한다.
3. **배경 이미지**: `build_scene_image_url(scene_en)` 재활용. scene_en이 없으면 직전 장면 이미지 URL 재사용.
4. 브로드캐스트: `{"type": "exploration_start", "place", "danger", "total": N, "image_url"}`

### 2-2. 탭 처리

새 WS 메시지 `explore_tap` (기존 `elif msg ==` 체인, `main.py:5343~`에 추가):

- 검증: 탐색 active / 플레이어 생존·방 소속 / **개인별 0.3초 쿨다운** (`exploration.last_tap_at[pid]`). 기존 3초 액션 레이트리밋은 **적용하지 않는다** (탭은 LLM 호출이 아니므로).
- `pos += 1` → 해당 칸 각본 조회 → 타입별 처리:

| 타입 | 처리 | 재활용 경로 |
|---|---|---|
| `flavor`/`empty` | 텍스트만 브로드캐스트 | — |
| `item` | **탭한 사람**에게 지급 | 합성 태그 문자열을 만들어 `parse_and_apply_items`에 통과 — 토스트·인벤·슬롯 분류가 기존과 동일하게 작동 |
| `gold` | **파티 전원** +amount | `parse_and_apply_gold` 경로 또는 동일 내부 함수 |
| `trap` | 탭한 사람 HP -damage, **최소 1 HP는 남김** (탐색으로 사망 금지) | 기존 HP 갱신·브로드캐스트 경로 |
| `enemy` | 탐색 **즉시 중단** → §2-3 | `[적 등장: 이름 | HP X]` 파이프라인 |

- 매 탭 브로드캐스트: `{"type": "explore_progress", "pos", "total", "tapper_name", "event": {...}}`

### 2-3. 종료 (3가지 경로)

1. **완주** (`pos == total`): `exploration_end {reason: "complete", summary}`. 방 메시지 히스토리에 시스템 노트 1줄 추가 — `[시스템] 탐색 완료: 폐허가 된 성채 — 획득: 녹슨 철검(허접), 골드 15` — **즉시 DM 호출은 하지 않는다** (토큰 절약). 다음 플레이어 행동 때 DM이 이 노트를 보고 이어간다.
2. **적 조우**: `exploration_end {reason: "enemy"}` → 곧바로 DM 턴 1회 강제 발동 (기존 행동 처리 경로에 "탐색 중 [이름]이(가) 굶주린 구울과 조우했다!"를 시스템 행동으로 주입) → DM이 서사 + `[적 등장]` 태그로 전투 개시. 기존 몬스터 시스템에 그대로 합류.
3. **방치 만료**: 마지막 탭 후 10분 경과 시 자동 종료 (기존 `_room_idle_sweeper`에 체크 1줄 추가 또는 다음 메시지 수신 시 lazy 체크). 방이 탐색 상태에 영구히 갇히는 일 방지.

### 2-4. 상태 지속성 (의도된 단순화)

- `room.exploration`은 **메모리 전용, 디스크 저장 제외** (`to_save_dict`에 넣지 않음). 저장/복원 사이에 탐색 중이었다면 그냥 증발 — DM이 다음 턴에 자연스럽게 이어감. ponytail: 세이브 마이그레이션 비용 대비 가치 없음.
- **재접속 복구는 지원**: `rejoin_ok` 페이로드에 진행 중 탐색 상태(place, pos, total, image_url) 포함 → 클라이언트가 UI 복원.

## 3. 클라이언트 (game.js / index.html / style.css)

- **탐색 오버레이**: 게임 화면 중앙 패널 (파티 패널·채팅은 유지). 구성: 배경 이미지(어두운 그라데이션) + 칸 단위 진행 게이지 + 큼직한 [🔍 탐색] 버튼 + 이벤트 팝 영역.
- **이벤트 팝 연출**: CSS keyframes로 카드가 톡 튀어나오는 애니메이션. 아이템=카드 형태, 골드=코인, 함정=흔들림+빨간 플래시, 적=실루엣 + 화면 진동. 라이브러리 추가 금지, 순수 CSS.
- 탭 후 0.3초 버튼 비활성(클라이언트에서도) + 사망/관전자는 버튼 숨김.
- 모든 플레이어가 브로드캐스트로 같은 게이지·연출을 본다. "허접이 녹슨 철검을 발견!" 처럼 탭한 사람 이름 표기.
- 이미지 로딩 전에는 실루엣/스피너.
- **[🔍 주변 탐색] 빠른 행동 버튼**: 시나리오 무관 상시 노출 버튼 1개 추가 (클라이언트 고정, 서버 변경 불필요). 동작은 일반 행동 전송: `"주변을 주의 깊게 탐색한다"`. 이게 플레이어발 탐색 진입로다.

## 4. 시스템 프롬프트 추가 (main.py DM 규칙 블록, ~948-1017 부근)

새 섹션 추가:

- 파티가 새 장소를 수색·탐험하려 하면 `[탐색: 장소명 | N칸 | 위험도 X]` 태그로 탐색 모드를 열어라. 응답 마지막 줄들 중 하나로.
- 칸수는 장소 규모에 맞게 6~16. 위험도는 서사 맥락에 맞게.
- **이미 뒤진 장소의 재탐색 요청은 태그 없이 서사로 거절하라** ("이미 샅샅이 뒤졌다").
- 전투 중에는 절대 열지 마라.
- 탐색 완료 후 `[시스템] 탐색 완료:` 노트가 히스토리에 보이면 그 결과를 이어받아 서사하라.

## 5. 구현하지 않는 것 (2단계 이후로 명시적 보류)

- 슬라이드 일러스트 여러 장 (1단계는 배경 1장 고정)
- 카드 수집첩/도감
- 적 조우 시 탐색 "재개" (중단으로 종결, 전투 후 DM이 새로 열면 됨)
- 탐색 상태 디스크 저장

## 6. 검증 체크리스트 (구현 후 전부 통과해야 완료)

1. `python -m py_compile main.py` / `node --check static/game.js`
2. 태그 파서 단위 체크: 정상형·칸수 생략·trailing space·전투 중 무시 — assert 스크립트 1개 (`_test_exploration.py`, 프레임워크 없이)
3. 각본 검증기 체크: 칸수 불일치 수리, enemy 2개→1개, JSON 깨짐→폴백
4. 실동 테스트: 서버 기동 → 방 생성 → (LLM 응답에 탐색 태그가 오도록 유도하거나 `_playtest_ws.py` 패턴으로 주입) → 탭 → 아이템 획득 토스트 → 인벤 반영 → 완주 → 시스템 노트 확인
5. 적 조우 경로: 탐색 중단 → `[적 등장]` → 몬스터 카드 노출 확인
6. 재접속: 탐색 중 새로고침 → UI 복원 확인
7. 서버 재시작 후 브라우저 강새로고침 (캐시버스터는 재시작 기반)
