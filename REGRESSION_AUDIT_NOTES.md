## 이번에 적용한 수정

### 1) 적 등장 태그 — DM 프롬프트 절대 규칙 추가
[main.py DM_SYSTEM_PROMPT](trog/main.py) — "🚨 절대 규칙" 섹션:
- 적이 시야에 들어온/적대 행동 표명한 그 응답 안에 **반드시** `[적 등장: 이름 | HP X | 속도 Y]` 발화
- "다음 턴에 찍지 뭐" 금지
- 구체 예시 ("고블린들이 돌진해 왔다" → 같은 응답에 3줄 등장 태그) 명시
- "적 카드 없이 서사로만 전투 묘사 = UI 빈 칸 → 플레이어 혼란" 강조

### 2) 탐색 → 아이템 보상 직접 지시
[main.py DM_SYSTEM_PROMPT](trog/main.py) — "🔍 탐색·수색 행동 → 아이템·정보 보상" 섹션 신설:
- 탐색/수색/조사 행동 시 **40~60% 확률로 보상 태그**
- 아이템 / 골드 / 단서 형식 명시
- 시체 노획 (전투 종료 후) 자동 보상 가이드
- "매번 '특별한 게 없다' 만 적기" 금지

### 3) 적 등장 토스트 알림 추가
[game.js showEventToasts](trog/static/game.js) — `events.monster_events` 의 `kind: 'spawn'` 처리:
- `👹 고블린 A 등장 (HP 12 ⚡14)` 토스트 (모바일에서도 즉시 보임)
- `kind: 'leave'` 도 추가 — `🌫 고블린 A 이탈/소멸`

이전엔 buff/debuff/tick/defeated 만 토스트, **spawn 은 누락**돼서 모바일 드로어 닫혀있으면 등장 자체를 모름. 이제 알림.

## 비판적 셀프 리뷰

### 🟡 LLM 이 새 프롬프트도 무시할 가능성
- 시스템 프롬프트가 이미 매우 길다 (1000줄+). 새 규칙 추가해도 LLM 이 묻힐 수 있음
- 특히 wrapper 모드 (Claude Code preset) — Anthropic 의 내부 프롬프트와 섞여 무시될 가능성
- 완화: 새 규칙을 "🚨 절대 규칙" 헤더 + 구체 예시로 강조했지만, 효과는 실제 플레이 후 검증 필요

### 🟡 탐색 보상 확률 40-60% 너무 높을 수도
- 맵의 모든 상자가 다 무언가를 줘야 한다는 인상 → 게임 단조로워질 위험
- 만약 매판 너무 자주 나오면 30%로 낮추거나 "흥미로운 단서" 비중을 늘려라
- 데이터: 실제 플레이 통계 후 조정

### 🔴 모바일에서 monster section 가시성
- party-panel 이 슬라이드 드로어 → 닫힌 상태에선 monster section 안 보임
- 신규 spawn 토스트로 알림은 되지만, 사용자가 토스트 사라진 뒤엔 또 안 보임
- **아직 미구현**: 게임 화면 상단에 "🟥 전투 중 (적 N) — 탭하여 보기" 같은 영구 인디케이터 추가 권장. 다음 작업 후보.

### 🟢 코드 레벨 회귀 없음
- 모든 V1~V4 기능 정상 존재 확인. 사용자 우려와 달리 "기능 빠짐"은 없음.
- 누락처럼 보이는 건 LLM 행동 이슈 → 프롬프트로 교정 가능.
